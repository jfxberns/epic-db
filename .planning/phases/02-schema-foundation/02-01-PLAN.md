Add three new functions to db_reader.py, placing them after the existing get_all_objects_from_msys() function and before the `if __name__` block. All code patterns are validated in 02-RESEARCH.md.

**1. get_column_metadata(db, table_name) -> list[dict]**

Access the AccessTable internals to extract per-column metadata. Import AccessTable and TableObj from access_parser_c.access_parser (AccessTable is already imported in _read_msys_objects_raw, so consolidate the import at module level).

For each column in AccessTable.columns:
- name: col.col_name_str
- type_code: col.type
- type_name: Map from TYPE_NAMES dict (1=Yes/No, 2=Byte, 3=Integer, 4=Long Integer, 5=Currency, 6=Single, 7=Double, 8=Date/Time, 9=Binary, 10=Short Text, 11=OLE Object, 12=Long Text (Memo), 15=GUID, 16=Decimal, 18=Complex)
- length_bytes: col.length
- max_chars: col.length // 2 for Text types (10, 12), else None (TEXT LENGTH IS BYTES not characters for UTF-16)
- can_be_null: col.column_flags.can_be_null
- is_autonumber: col.column_flags.autonumber
- is_fixed_length: col.column_flags.fixed_length
- column_id: col.column_id (needed for index column mapping)
- column_index: col.column_index

For Text columns (type 10, 12): also extract collation from col.various if present.
For Decimal columns (type 16): also extract precision and scale from col.various if present.

The table access pattern: db.catalog.get(table_name) -> page_num, page_num * db.page_size -> table_offset, db._tables_with_data.get(table_offset) or fall back to db._table_defs.get(table_offset) wrapped in TableObj. Then construct AccessTable and iterate .columns. See 02-RESEARCH.md Pattern 1 for the exact validated code.

**2. get_relationships(db) -> list[dict]**

Read MSysRelationships at offset 5 * page_size using the same AccessTable pattern as _read_msys_objects_raw(). Parse the szRelationship, szObject, szColumn, szReferencedObject, szReferencedColumn, and grbit columns. For each row, compute integrity flags from grbit bitmask:
- 0x001: ENFORCE_REFERENTIAL_INTEGRITY
- 0x002: CASCADE_UPDATES
- 0x004: CASCADE_DELETES
- 0x100: NO_INTEGRITY
- 0x1000: INHERITED
- 0x2000000: QUERY_BASED

Return list of dicts with: name, source_table, source_column, ref_table, ref_column, grbit, enforce_ri (bool), cascade_updates (bool), cascade_deletes (bool), no_integrity (bool). Strip null bytes from all string values.

See 02-RESEARCH.md Pattern 2 for the exact validated code.

**3. get_indexes_for_table(db, table_name, column_id_map) -> list[dict]**

Parse binary REAL_INDEX2, ALL_INDEXES, and INDEX_NAME structures from the table definition bytes. This requires:
- Import construct structs: Struct, Array, Int8ul, Int16ul, Int32ul, PaddedString from construct
- Import parse_table_head, TDEF_HEADER from access_parser_c.parsing_primitives
- Navigate past real_index entries (12 bytes each), column entries (25 bytes each), and column_names (variable length) in the merged table data to reach REAL_INDEX2
- Parse REAL_INDEX2 (52 bytes each, contains col_id->index mapping via unk_struct with 10 slots, unused slots have col_id=0xFFFF)
- Parse ALL_INDEXES (28 bytes each, contains idx_type: 0=Regular, 1=Primary Key, 2=Foreign Key)
- Parse INDEX_NAME (variable length, UTF-16 encoded names)
- Handle multi-page table definitions (follow next_page_ptr chain)

The column_id_map parameter maps column_id -> column_name (build it from get_column_metadata() results before calling this function).

Return list of dicts with: name, type (Primary Key/Foreign Key/Regular), columns (list of column names).

See 02-RESEARCH.md Pattern 3 for the exact validated code.

Also add a helper function strip_null_bytes(s) that returns s.replace('\x00', '') if s is a string, else str(s) -- consolidating the null byte stripping pattern from validate_encoding.py into db_reader.py for reuse.

Move the AccessTable import from inside _read_msys_objects_raw() to the module-level imports, and add TableObj to the same import. Add the construct imports at module level too.

  <name>Task 2: Extract relationships and produce assessment/relationships.md</name>
  <files>scripts/extract_relationships.py, assessment/relationships.md</files>
  <action>
Create scripts/extract_relationships.py that imports from db_reader and produces assessment/relationships.md.

The script should:
1. Import open_db, get_relationships, get_user_tables from db_reader
2. Call get_relationships(db) to get all 14 MSysRelationships entries
3. Get user table list to classify relationships as table-to-table vs query-based
4. Separate into three categories:
   a. **Table-to-Table Relationships** -- both source and referenced are user tables
   b. **Table-to-Query Relationships** -- one side references a query (szObject or szReferencedObject contains "qry" or is not in user tables list)
   c. **System Relationships** -- names starting with "MSysNavPane"
5. For each relationship, strip null bytes from all string values
6. Format output as markdown with tabulate (tablefmt="pipe")

The assessment/relationships.md document structure:
```
# Database Relationships

## Summary
- {N} total relationships in MSysRelationships
- {N} table-to-table relationships (user-defined foreign keys)
- {N} table-to-query relationships (lookup/derived)
- {N} system relationships (excluded from analysis)

## Table-to-Table Relationships

| Relationship Name | Source Table | Source Column | Referenced Table | Referenced Column | Integrity |
|...table rows...|

### Referential Integrity Detail

For each relationship with enforced integrity or cascade rules, add a bullet explaining what it means:
- "{name}": {source_table}.{source_column} -> {ref_table}.{ref_column} -- {flags description}

## Table-to-Query Relationships

| Relationship Name | Source | Source Column | Referenced Query/Table | Referenced Column | Flags |
|...table rows...|

Note: These relationships reference queries rather than tables directly. The actual table dependencies are resolved through the query definitions (Phase 3).

## System Relationships

Excluded: {list of system relationship names}

## Notes

- grbit=0 means no referential integrity enforced (relationship exists for UI lookup purposes only)
- Relationships flagged NO_INTEGRITY (0x100) allow orphaned records
- QUERY_BASED (0x2000000) relationships depend on query definitions for resolution
- Default values and validation rules are stored in field property blobs (not extractable on macOS -- deferred to Phase 3 Windows extraction if needed)
```

Use tabulate with tablefmt="pipe" for markdown tables. Strip null bytes from all string values using the strip_null_bytes helper from db_reader.

The script should print progress to stdout and write the markdown file.
