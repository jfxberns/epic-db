**File 1: windows/export_all.vbs**

A complete, robust VBScript for headless Access object export. Key requirements:

1. **Arguments:** Takes 2 args: `dbPath` (path to .accdb) and `outputPath` (export root directory)
2. **Directory creation:** Creates subdirectories under outputPath: forms/, reports/, queries/, queries_sql/, macros/, modules/
3. **Form export (SaveAsText acForm=2):** Iterate `db.Containers("Forms").Documents`, call `accessApp.SaveAsText 2, doc.Name, outputPath & "\forms\" & doc.Name & ".txt"` for each. Log each form name to stdout.
4. **Report export (SaveAsText acReport=3):** Same pattern with `db.Containers("Reports").Documents` and type constant 3.
5. **Query export (SaveAsText acQuery=1):** Iterate `db.QueryDefs`, skip queries starting with "~" (temp system queries). For each:
   - Export SaveAsText definition: `accessApp.SaveAsText 1, qdf.Name, ...`
   - Export raw SQL to queries_sql/ as .sql file: `fso.CreateTextFile(..., True, True).Write qdf.SQL`
   - Log query name and type to stdout
6. **ALSO export `~sq_c*` queries separately:** These 8 hidden subqueries are subform data sources. Export them to a queries_sql/subqueries/ subdirectory. Do NOT skip them.
7. **Macro export (SaveAsText acMacro=4):** Iterate `db.Containers("Scripts").Documents`, skip names starting with "~".
8. **Module export (SaveAsText acModule=5):** Iterate `db.Containers("Modules").Documents`.
9. **Error handling:** Wrap each individual export in `On Error Resume Next` / `On Error GoTo 0` so one failed object does not abort the entire export. Log errors to stderr.
10. **Cleanup:** Close database and quit Access at the end.
11. **Unicode output:** Use `CreateTextFile(path, True, True)` for Unicode (UTF-16-LE) output for the SQL files.
12. **Access visibility:** Set `accessApp.Visible = False` for headless operation.

**File 2: windows/export_all.ps1**

PowerShell wrapper that:
1. Takes optional parameters: `-DbPath` (default: ".\epic_db.accdb"), `-OutputPath` (default: ".\export")
2. Validates the database file exists before calling the VBScript
3. Creates the output directory if needed
4. Runs: `cscript //nologo "$PSScriptRoot\export_all.vbs" $DbPath $OutputPath`
5. Reports elapsed time and summary (count of exported files by type)
6. Lists any errors from the VBScript output

**File 3: windows/README.md**

Step-by-step instructions covering:

1. **Windows Environment Options:**
   - Option A: UTM VM on macOS (free) -- link to https://docs.getutm.app/guides/windows/, note Windows 11 ARM ISO from Microsoft
   - Option B: Cloud Windows instance (Azure, AWS) -- brief note
   - Option C: Physical Windows machine with Access

2. **Prerequisites:**
   - Microsoft Access (Microsoft 365 subscription with Access, or standalone Access 2019+)
   - On ARM64 Windows: Microsoft 365 has native ARM64 Access builds. Standalone Access may need x86 emulation (transparent on Win11 ARM).

3. **Setup Steps:**
   - Copy epic_db.accdb to the Windows machine (e.g., C:\epic_db\epic_db.accdb)
   - Copy the windows/ folder to the same location (e.g., C:\epic_db\windows\)
   - Open PowerShell, navigate to C:\epic_db\windows\
   - Run: `.\export_all.ps1 -DbPath "C:\epic_db\epic_db.accdb" -OutputPath "C:\epic_db\export"`
   - Wait for completion (expect 30-60 seconds for 74 objects)

4. **Thai Encoding Note:**
   - If Windows locale is English, Thai characters in form captions MAY be garbled
   - Mitigation: In Windows Settings > Time & Language > Language, add Thai language pack
   - Or: Enable "Beta: Use Unicode UTF-8 for worldwide language support" in Region settings
   - Test: Open one exported form .txt file and verify Thai text is readable

5. **Bringing Files Back:**
   - Copy the entire export/ directory back to macOS
   - Place it at: /Users/jb/Dev/epic_gear/epic-db/windows/export/
   - The directory should contain: forms/, reports/, queries/, queries_sql/, macros/, modules/

6. **Expected Output:**
   - 17 .txt files in forms/
   - 25 .txt files in reports/
   - ~24 .txt files in queries/ (visible queries) + ~24 .sql files in queries_sql/
   - ~8 .sql files in queries_sql/subqueries/ (hidden subqueries)
   - 0 files in macros/ (no macros exist)
   - 0 files in modules/ (no modules exist)

7. **Troubleshooting:**
   - "ActiveX component can't create object" -> Access is not installed
   - "Could not find installable ISAM" -> Access database engine not registered
   - Script hangs -> Access may have opened a dialog; kill MSACCESS.EXE and retry with fresh database copy
