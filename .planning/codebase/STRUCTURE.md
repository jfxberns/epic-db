## Directory Purposes

**`.claude/`**
- Purpose: Claude Code editor configuration and permission settings
- Contains: JSON configuration files for the project
- Key files: `settings.local.json` - defines allowed skills (gsd:map-codebase)

**`.planning/`**
- Purpose: GSD (Goal, Steps, Decisions) phase planning and documentation artifacts
- Contains: Codebase analysis documents generated by mapping tools
- Key files: (generated ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md, STACK.md, INTEGRATIONS.md as needed)

**`.planning/codebase/`**
- Purpose: Central repository for codebase analysis and reference documents
- Contains: Analysis of technology stack, architecture, structure, conventions, testing patterns, and technical concerns
- Generated by: `/gsd:map-codebase` command with various focus areas

**`.venv/`**
- Purpose: Python virtual environment for project dependencies
- Contains: Python 3.11 packages and activation scripts
- Note: May contain extraction/analysis scripts for parsing the Access database

**`data/`**
- Purpose: Project data storage and database files
- Contains: Microsoft Access database and separate git tracking
- Key files: `epic_db.accdb` - the main system under assessment (10MB, contains all business logic and data)
- Separate `.git/`: Enables independent versioning of data files

**`docs/`**
- Purpose: Project documentation and context
- Contains: Overview, requirements, assessment goals
- Key files: `overview.md` - Brief project description, current system details, pain points, and deliverables

## Naming Conventions

**Files:**
- Main database: `epic_db.accdb` (Access database binary format)
- Documentation: `*.md` (Markdown format)
- Configuration: `settings.local.json` (JSON format)

**Directories:**
- Hidden configuration: `.claude/`, `.git/`, `.planning/` (leading dot indicates internal/meta)
- Functional directories: `data/`, `docs/` (lowercase, clear purpose)

**Database Objects (within Access):**
- Tables: Likely English names (Customers, Orders, Products, Inventory)
- Forms: Action-oriented names (e.g., "Order Entry", "Customer Management")
- Reports: Descriptive names (e.g., "Inventory Status", "Invoice", "Shipping Labels")
- VBA Modules: Functional names (e.g., "Pricing", "Discounts", "Formulas")
- Note: User interface in Thai language despite English database structure

## Special Directories

**`.git/` and `data/.git/`:**
- Purpose: Dual git tracking - main repo tracks project files, data repo tracks database changes separately
- Generated: Yes (git initialization)
- Committed: Git metadata (to separate repos)
- Rationale: Allows independent versioning of large binary `.accdb` file

**`.venv/`:**
- Purpose: Python virtual environment (isolated package management)
- Generated: Yes
- Committed: No (typically in .gitignore)
- Rationale: Assessment may use Python scripts to extract/analyze Access database structure

**`.claude/`:**
- Purpose: Claude Code editor project settings
- Generated: Yes (by Claude environment)
- Committed: Yes (project configuration)
- Contains: Permission definitions for available GSD skills
