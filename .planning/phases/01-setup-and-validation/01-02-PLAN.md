  <name>Task 1: Create inventory script and generate assessment/inventory.md</name>
  <files>scripts/inventory.py, assessment/inventory.md, assessment/tables/.gitkeep, assessment/queries/.gitkeep, assessment/forms/.gitkeep, assessment/reports/.gitkeep, assessment/modules/.gitkeep, assessment/macros/.gitkeep</files>
  <action>
1. Create the assessment directory structure:
   ```
   assessment/
   ├── inventory.md
   ├── tables/
   ├── queries/
   ├── forms/
   ├── reports/
   ├── modules/
   └── macros/
   ```
   Add `.gitkeep` files in empty subdirectories so they are tracked by git.

2. Create `scripts/inventory.py` with these capabilities:

   a. **Import from db_reader**: Use `open_db`, `get_all_objects_from_msys`, `get_catalog`, `parse_table`.

   b. **Table inventory**: For each user table (Type 1, non-MSys):
      - Get row count by reading the table and counting rows (use `len(list(parse_table(db, name).values())[0])` or similar — access-parser returns columns as lists)
      - Get column count
      - Store: name, row_count, column_count

   c. **Query inventory**: Read MSysObjects for Type 5 objects. For each query:
      - Get the query name
      - Attempt to determine query type: try reading MSysQueries table, match on query name. The MSysQueries table has an Attribute column — flag value 0=Select, 16=Crosstab, 32=Delete, 48=Update, 64=Append, 80=MakeTable, 96=DataDef, 112=Passthrough, 128=Union (from Research QUERY_FLAGS). If MSysQueries is unreadable or type cannot be determined, default to "Unknown".
      - Store: name, query_type

   d. **Form/Report/Macro/Module inventory**: Read MSysObjects for Types -32768 (Form), -32764 (Report), -32766 (Macro), -32761 (Module). For each:
      - Get the name
      - Note: content extraction requires Windows (per research — these are binary blobs)
      - Store: name, type

   e. **Windows assessment logic**: After inventory, determine:
      - Can macOS list form names? (yes, from MSysObjects)
      - Can macOS list report names? (yes, from MSysObjects)
      - Can macOS extract form content/layout? (no — binary blobs)
      - Can macOS extract VBA code? (no — binary blobs)
      - Recommendation: If forms, reports, or modules exist → Windows needed for content extraction. If none exist → Windows not needed.

   f. **Generate assessment/inventory.md** as a Markdown document with:
      - Header with generation timestamp and database file path
      - Summary section: total count per object type in a table
      - Tables section: markdown table with Name, Row Count, Column Count for each table
      - Queries section: markdown table with Name, Query Type for each query
      - Forms section: list of form names (with note that content requires Windows)
      - Reports section: list of report names (with note that content requires Windows)
      - Modules section: list of module names (with note that content requires Windows)
      - Macros section: list of macro names (with note that content requires Windows)
      - Windows Assessment section: clear YES/NO recommendation with reasoning
      - Use tabulate library for formatting markdown tables

   g. **Make it rerunnable**: The script should accept an optional `--output` argument for the output path (default: `assessment/inventory.md`). Running it again overwrites the previous inventory. Print the output path when done.

3. Run the script: `python scripts/inventory.py`

4. Print the generated `assessment/inventory.md` to stdout for review.

2. **Check for linked tables**: Look for MSysObjects Type 4 (Linked ODBC) and Type 6 (Linked). If any exist, add a "Linked Tables" section to inventory.md listing them with their link source if available (check for a Connect or Database column in MSysObjects).

3. **Check for other/unknown types**: Scan MSysObjects for any Type values not in the MSYS_TYPES mapping. If found, add an "Other Objects" section listing them with their raw type codes.
