**Step 2: Create `scripts/db_reader.py`** -- the centralized database access module all scripts will import.

Constants and imports:
- `DB_PATH = Path(__file__).parent.parent / "data" / "epic_db.accdb"` (resolve relative to script location)
- Import `AccessParser` from `access_parser`

Functions to implement:
- `open_db(db_path=DB_PATH) -> AccessParser`: Opens the database. Wrap in try/except; on failure print clear message about what went wrong (version mismatch, file not found, import error).
- `get_catalog(db) -> list[str]`: Returns `list(db.catalog.keys())` -- user-visible table names only. Note: catalog filters to Type=1 only per research Pitfall 2.
- `parse_table(db, table_name) -> dict`: Wraps `db.parse_table(table_name)`. Returns dict of `{column_name: [values...]}`.
- `get_all_objects_from_msys(db) -> dict`: Reads MSysObjects via `db.parse_table("MSysObjects")`. Groups objects by type using this mapping:
  ```python
  MSYS_TYPES = {
      1: "Table (Local)",
      4: "Table (Linked ODBC)",
      5: "Query",
      6: "Table (Linked)",
      -32768: "Form",
      -32764: "Report",
      -32766: "Macro",
      -32761: "Module",
  }
  ```
  Filters out names starting with "MSys" or "~" (system/temp objects). Returns `{"Table (Local)": [{"name": "...", "flags": N}, ...], ...}`. Include an "Unknown" bucket for type codes not in the mapping.

Add `if __name__ == "__main__"` block that:
1. Opens the database
2. Prints catalog table names with count
3. Prints MSysObjects type summary (type label -> count)
4. Reads the first table from catalog, prints first 3 rows using tabulate with `tablefmt="pipe"`

Run `python scripts/db_reader.py` to verify. If it fails:
- ImportError -> check venv activation, re-install
- "not a supported database version" -> file may need McSash fork (`pip install git+https://github.com/McSash/access_parser_c`), update import accordingly
- FileNotFoundError -> check DB_PATH resolution

**Step 3: Create `scripts/validate_encoding.py`** -- Thai encoding validation script.

This script must:

a. **Import from db_reader** (add `scripts/` to sys.path if needed, or use relative import).

b. **Check column names across ALL tables**: For every table in catalog, get column names. Check each for Thai characters (any char in range U+0E00-U+0E7F). Print all Thai-containing column names grouped by table.

c. **Check row data across ALL tables**: For every table, read all rows. For every string value:
   - Count values containing Thai characters
   - Count values containing U+FFFD replacement characters (mojibake indicator)
   - Count values containing null bytes \x00 (encoding corruption indicator)
   Print summary per table: total strings scanned, Thai count, mojibake count, corruption count.

d. **Print visual samples**: For each table containing Thai data, print 3-5 sample Thai values with format:
   ```
   TABLE_NAME.COLUMN_NAME[row_index]: "actual Thai text here"
   Unicode codepoints: U+0E01 U+0E02 ...
   ```
   This is what the user (or someone who reads Thai) will visually verify.

e. **Print clear verdict**:
   - `PASS`: Thai characters found, zero mojibake, zero corruption
   - `FAIL`: Mojibake or corruption detected. Print affected tables/columns. Suggest: "Try McSash fork: pip install git+https://github.com/McSash/access_parser_c"
   - `WARNING`: No Thai characters found anywhere. Flag as unexpected.

f. **Handle encoding failure**: If verdict is FAIL:
   1. Install McSash fork: `uv pip install git+https://github.com/McSash/access_parser_c`
   2. Update db_reader.py to import from access_parser_c instead of access_parser
   3. Re-run validate_encoding.py
   4. If STILL broken: document as a hard blocker per user decision. Do NOT proceed.

Run `python scripts/validate_encoding.py` and capture the full output.
