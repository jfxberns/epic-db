<task type="auto">
  <name>Task 2: Create cross-table data profiling and abandoned table analysis</name>
  <files>scripts/extract_profiles.py, assessment/data_profile.md</files>
  <action>
Create scripts/extract_profiles.py that produces assessment/data_profile.md with cross-table profiling and abandoned table detection.

The script should:
1. Import open_db, get_user_tables, parse_table, get_relationships from db_reader
2. For each user table, compute:
   - row_count: number of rows
   - column_count: number of columns
   - Per-column fill rate: (non-null, non-empty values) / total rows
   - Average fill rate across all columns
   - Distinct value count estimate for each column (use set on first 1000 values)
3. Cross-reference with relationships:
   - Call get_relationships() to get all relationships
   - For each user table, check if it appears as source or reference in any relationship
   - Tables with 0 relationships AND 0 rows are strong abandoned signals
4. Compute abandoned table signals per table:
   - ZERO_ROWS: row_count == 0
   - VERY_LOW_FILL: average fill rate < 10% and row_count > 0
   - NO_RELATIONSHIPS: table not in any relationship (source or reference)
   - Tables with 2+ signals = "Likely Abandoned", 1 signal = "Review", 0 signals = "Active"

**assessment/data_profile.md structure:**

```markdown
# Data Profile Summary

## Overview

| Table | Rows | Columns | Avg Fill Rate | Relationships | Status |
|-------|------|---------|---------------|---------------|--------|
| {table_name} | {rows} | {cols} | {avg_fill%} | {N in / N out} | {Active/Review/Likely Abandoned} |

Total: {sum rows} rows across {N} tables

## Abandoned Table Analysis

### Likely Abandoned

{For each table with 2+ signals:}
#### {table_name}
- **Signals:** {list of signals}
- **Evidence:** {explanation}
- **Recommendation:** {e.g., "Safe to exclude from rebuild -- empty table with no relationships"}

### Under Review

{For each table with 1 signal:}
- **{table_name}**: {signal} -- {brief explanation}

### Active Tables

All remaining tables show healthy data patterns (non-zero rows, relationships present, reasonable fill rates).

## Per-Table Column Profiles

### {table_name} ({N} rows)

| Column | Fill Rate | Distinct Values | Sample Values |
|--------|-----------|-----------------|---------------|
| {col} | {fill%} | {distinct_count} | {3 sample values, truncated} |

{Repeat for each table}

## Notes

- Fill rate = (non-null, non-empty values) / total rows
- Distinct values estimated from first 1000 rows where applicable
- "Relationships" count includes both incoming (referenced by other tables) and outgoing (references other tables)
- Abandoned detection uses multiple signals -- single-signal tables may still be active
```

**Important implementation details:**
- Strip null bytes from text sample values
- For bytes-type values, count as non-empty but show "[Binary]" in samples
- For columns where all values are None or empty string, fill rate = 0%
- Relationship counts: "N in" = times table appears as ref_table, "N out" = times as source_table
- Format fill rates as percentages with 0 decimal places
- Use tabulate for all tables
- Print progress: "Profiling: {table_name}... ({N} rows, avg fill {X}%)"
