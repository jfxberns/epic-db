**Step 2: Create `scripts/inventory.py`.**

Import from db_reader (add scripts/ to sys.path or use the same pattern as validate_encoding.py from Plan 01). Use `open_db`, `get_catalog`, `parse_table`, `get_all_objects_from_msys`.

The script must implement these functions:

a. `get_table_metadata(db) -> list[dict]`: For each user table from get_catalog(db):
   - Parse the table to get row count: `len(next(iter(table_data.values())))` if table_data has columns, else 0
   - Get column count: `len(table_data.keys())`
   - Handle parse errors gracefully: if a table fails to parse, record row_count=-1 (error) and continue
   - Return list of `{"name": str, "row_count": int, "column_count": int}`

b. `get_query_metadata(db) -> list[dict]`: From get_all_objects_from_msys(db), take the "Query" entries. For query type classification:
   - Try reading MSysQueries via `db.parse_table("MSysQueries")`
   - If readable, for each query find rows where Attribute==0 (query definition row). The Flag value on that row encodes query type: 0=SELECT, 16=CROSSTAB, 32=DELETE, 48=UPDATE, 64=APPEND, 80=MAKE_TABLE, 96=DDL, 112=PASSTHROUGH, 128=UNION
   - If MSysQueries is unreadable, fall back to the Flags field from MSysObjects entries (research notes this may also encode type for Type=5 objects)
   - If neither works, set type to "Unknown"
   - Return list of `{"name": str, "query_type": str}`

c. `get_other_objects(db) -> dict`: From get_all_objects_from_msys(db), collect forms, reports, macros, modules by name. Also check for:
   - Linked tables (Type 4 or Type 6): if any exist, include them separately with a note about external data source
   - Unknown types: any MSysObjects Type not in MSYS_TYPES mapping
   - Return `{"forms": [...], "reports": [...], "macros": [...], "modules": [...], "linked_tables": [...], "unknown": [...]}`

d. `assess_windows_need(other_objects) -> str`: Generate Windows assessment text:
   - If forms, reports, OR modules exist: "Windows environment IS NEEDED for content extraction. macOS can list names (shown above) but cannot extract form layouts, report definitions, or VBA code. These are stored as binary blobs requiring Microsoft Access COM Object Model (Application.SaveAsText)."
   - If none of those exist: "Windows environment is NOT needed. No forms, reports, or modules were found in the database."
   - Per user decision: macOS listing names = viable for Phase 1 inventory. Content extraction deferred to Windows.

e. `generate_inventory(db, output_path) -> None`: Orchestrates all the above and writes assessment/inventory.md. Use tabulate with `tablefmt="pipe"` for markdown tables. Document structure:

```markdown
# Epic DB Object Inventory

**Generated:** {date}
**Source:** data/epic_db.accdb (Access 2010 format)
**Generator:** scripts/inventory.py (rerun to regenerate)

## Summary

| Object Type | Count |
|---|---|
| Tables | X |
| Queries | Y |
...

Total: N objects

## Tables (X)

| Name | Row Count | Columns |
|---|---|---|
...

## Queries (Y)

| Name | Type |
|---|---|
...

## Forms (Z)

| Name |
|---|
...

> Note: Form content (layouts, controls, VBA code-behind) requires Windows with Microsoft Access for extraction.

## Reports (W)
[same pattern]

## Modules (V)
[same pattern]

## Macros (U)
[same pattern]

## Linked Tables
[only if any exist, with note about external data source]

## Windows Assessment

[output from assess_windows_need]
```

f. **CLI interface**: Accept optional `--output` argument (default: `assessment/inventory.md`). Print final summary to stdout:
